In­num­er­able E­numer­ables

Many Enumerable methods return the same data type as their receiver, in a form that echoes the original object one-for-one. Essentially, they’re like a filter that returns a processed version of the original object. #group_by is an interesting deviation from that pattern. It returns a whole new hash. You can think of it as generating a report on the object.

#group_by still runs the supplied code block on each element of the object. But instead of giving you back essentially the same object, with the results in place of the original values, it gives you a hash of the original values grouped together based on which results they produced. In other words, if five elements all produced a result of 3, you’ll find one hash entry for 3, associated with a list of those five elements.

Let’s walk through it from the beginning. Here we have an array. We’re going to call #group_by on it and give it some code.

my_array = [ “cat”, “dog”, “horse”, “dolphin”, “coat”, “drive”, “duck”, “David”] my_array.group_by { |item| item[0].downcase }

So what happens here?

Like any other iterator, #group_by goes through the array one element at a time, plugging its value into the code block and running it. But instead of giving you back the results right away, it hangs onto them, tallying up which elements give it the same answer.

In this case, the code block takes each element and returns the first letter converted to lowercase. This is what it gets:

[ “cat”, “dog”, “horse”, “dolphin”, “coat”, “drive”, “duck”, “David”] —> [“c”, “d”, “h”, “d”, “c”, “d”, “d”, “d”]

Now it groups together the elements that got the same results:

c: cat, coat d: dog, dolphin, drive, duck, David h: horse

and returns this information to you as a hash of key-value pairs.

{ “c” => [“cat”, “coat”] “d” => [“dog”, “dolphin”, “drive”, “duck”, “David”] “h” => [“horse”] }

This can be useful when you’re trying not just to manipulate the values in the array, but to make a determination about the array as a whole.
