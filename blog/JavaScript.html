So here is a (very) quick primer on a couple different ways to iterate through collection objects in Ruby vs. JavaScript.

Imagine iterating through, say, an array in Ruby. You can do a for loop:

for i in 0..array.length { do_something (array[i]) }

Here, i stands for a number, which you plug into array as an index to access the element you need.

But that's annoying when you also have enumerators and can just do this:

array.each {|x| x.do_something}

Here, the element itself from array gets passed into x. You can take out the middleman of having to call it from the array by its index. And it's clearer to the reader that what we want to deal with here is each item in array, not a bunch of numbers, which are just the means to an end.

Does JavaScript have an easy built-in way to iterate through a range of numbers like (0..10)? Does it have anything like the each method? I don't know! All I know is you can iterate through an array using this amazingly tedious for loop:

for (i = 0; i < array.length; i++) { array[i].doSomething; }

or this slightly-less-tedious shorthand.

for (i in array) { array[i].doSomething; }

Note here that i does not get the array element. Just its index. We still have to reference the element from the array! If you try this:

for (i in array) { i.doSomething; }

you're in for a headache.

It took me a while before I realized that's what was happening. It doesn't look like that's what it's supposed to mean! You'd think "i in array" would mean each thing in array, meaning each element. If you wanted to iterate through the indices you'd think the phrase would be something like "i in array.indexes" or "i in (0..array.length)"!

Well, there's JavaScript for you.
